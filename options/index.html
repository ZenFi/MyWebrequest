Block
://
域名应遵守以下规则:
  • 可以指定确切的域名, 如 www.abc.com
  • 也可以使用IP地址, 如 8.8.8.8
  • 你可以使用通配符 * 来匹配任意子域名, 但只能在二级及之上的域名中使用. 如 *.abc.com 和 *.api.abc.com 是允许的, 而诸如 *.com 和 *.net 则是非法的.
/
路径应遵守以下规则(请省略路径开头的 /):
  • 可以指定确切的路径, 如 path, path?num=1
  • 你可以使用通配符 * 来模糊匹配路径, 如:
    • *(即任意路径, 包括空)
    • path*(所有以 path 开头的路径)
    • path*.jsp(以 path 开头并以 .jsp 结尾的路径)
    • *kwd*(任何包含 kwd 的路径)
    • path?*(所有提交到 path 的query请求)
    • path?num=*(所有提交path的query请求, 且num是第一个请求参数，注意 path?ps=23&num=23 不会被匹配)
://
匹配URL应遵守以下规则:
  • 可以使用{kwd}的方式来命名你所关注的URL部分, 你可以在跳转规则中使用他们, 如 {name}.abc.com/, uk.abc.com/news/{newsid}-{time}.html
  • URL必须指定一级域名, {fld}.com/ 规则是不允许的
  • 可以使用{*kwd}的方式来匹配路径中末尾的部分, query字符串不会被匹配. 如 uk.abc.com/news/{*newspath} 能匹配 uk.abc.com/news/intl/887.html (此时newspath代表 intl/887.html)
  • 你也可以用{kwd}的方式来命名query字符串中你所关注的值. 如 www.google.com.hk/search?q={kwd} 能匹配你在谷歌中搜索的关键字
  • 扩展已经预定义了以下的名字, 你自定义的名字不能和下面的重复:
    • p 代表协议如 http, https
    • h 代表URL中的域名, 如 g.cn, uk.abc.com
    • m 代表URL中的一级域名, 对于 g.cn/search* 则代表 g.cn, 对于 uk.abc.com/news/*则代表 abc.com
    • r 代表URL中的路径(不包含开头的/), 如 search, news/998.html
    • q 代表URL中的query字符串(不包含开头的的?), 如 newwindow=1&safe=2
跳转URL应遵守以下规则:
  • 可以使用在匹配URL中定义的名字
  • 也可以使用扩展预定义的:
    • p 代表协议如 http, https
    • h 代表URL中的主机名(域名), 如 g.cn, uk.abc.com
    • m 代表URL中的一级域名, 对于 g.cn/search* 则代表 g.cn, 对于 uk.abc.com/news/*则代表 abc.com
    • r 代表URL中的路径(不包含开头的/), 如 search, news/998.html
    • q 代表URL中的query字符串(不包含开头的的?), 如 newwindow=1&safe=2
